# Project Context – OCR Localization Checker

## Общая цель
Создать инструмент автоматизации для тестирования локализации маркетинговых баннеров. Пользователь загружает ZIP‑архив с изображениями и текстами‑эталонами. Система должна:
1. Извлечь текст с изображения (многослойный OCR: Tesseract → PaddleOCR → Google Vision).
2. Сравнить OCR‑результат с эталонным текстом.
3. Отчётливо сообщить о совпадении/несовпадении и позволить ручную проверку при неуверенности.

## Среда деплоя
- Google Cloud Run
- Python 3.10‑slim в контейнере
- Автодеплой через GitHub → Cloud Build → Artifact Registry → Cloud Run  
- Никаких VPS/десктоп‑вариантов, только контейнер.

## OCR‑стратегия
1. **Tesseract** — основной, быстрый и бесплатный.
2. **PaddleOCR** — fallback при плохом качестве.
3. **Google Vision** — арбитр для окончательного решения. Использовать только после неудачи первых двух.

## Ограничения
- Память контейнера: 512 МБ – 1 ГБ.
- Не использовать платные сервисы без явного разрешения.
- Поддерживать несколько языков.
- Docker‑образ должен собираться на бесплатном уровне Cloud Run.

## Роль AI‑разработчика
Перед написанием кода **обязательно**:
1. Проверить системные зависимости (apt‑пакеты).
2. Оценить потребление памяти и совместимость с Cloud Run.
3. Обозначить риски (например, большой объём памяти у PaddleOCR) и предложить варианты.
4. Обновить документацию (README) и добавить тесты.

Каждый Pull Request должен обновлять `AI_PROGRESS.md` и не нарушать архитектуру без явного запроса.
